Code Review Agent

面向 GitHub Pull Request 的智能代码评审系统。自动解析 Diff，结合 RAG 规范与历史经验，完成多维度审查并生成结构化报告与 PR 评论。

01 · 问题背景

人工 Code Review 成本高、反馈慢。只把 Diff 贴给大模型的方式存在上下文不足、规范不一致、token 失控与无法接入协作流程等问题。需要的是可工程化的审查闭环，而不是一次性 Demo 脚本。

02 · 解决方案

GitHub PR Event → Webhook 校验 → Diff 解析 / 语言过滤 → 分块 & 增量审查（控 token） → RAG：规范 / 历史 Review / 最佳实践 → LLM 多维度评分与建议 → 结构化报告 + PR Comment

03 · 我的贡献

  • 设计审查流水线：Diff 解析、语言过滤、分块增量策略。
  • 实现 RAG 增强：规范文档、历史评审与最佳实践检索。
  • 打通 Webhook / API / CLI，支持本地调试与服务化部署。
  • 输出结构化报告模板与 PR 评论，进入真实协作流。

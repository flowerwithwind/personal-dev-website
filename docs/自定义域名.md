# 绑定自定义域名（GitHub Pages）

当前默认地址：

`https://flowerwithwind.github.io/personal-dev-website/`

绑定自己的域名后，可使用例如：

`https://www.你的域名.com/`  
`https://你的域名.com/`

路径如 `/projects/` 会变成：`https://www.你的域名.com/projects/`（内容不变，只是主机名变漂亮）。

---

## 你需要准备

1. 已购买的域名（任意注册商：阿里云、腾讯云、Cloudflare、Namecheap 等）
2. 能改该域名的 **DNS 解析**
3. 把域名告诉开发配置（或自己改 `site/domain.config.mjs`）

---

## 一、改站点配置

编辑 `site/domain.config.mjs`：

```js
export const customDomain = 'www.你的域名.com'; // 不要写 https://
```

推荐优先用 **www 子域**（配置更简单）。若坚持用根域名 `你的域名.com`，见下方 DNS 说明。

然后：

```bash
cd site
npm run build
powershell -ExecutionPolicy Bypass -File .\deploy-gh-pages.ps1
```

构建会自动把域名写入 `dist/CNAME`，GitHub Pages 靠它绑定域名。

---

## 二、在 GitHub 仓库开启 Pages 自定义域名

1. 打开 https://github.com/flowerwithwind/personal-dev-website/settings/pages  
2. **Custom domain** 填与 `domain.config.mjs` **完全相同**的域名  
3. 保存后勾选 **Enforce HTTPS**（证书签发可能要几分钟到几小时）

---

## 三、DNS 解析（在域名服务商后台）

### 方案 A：www 子域（推荐）

| 类型 | 主机记录 | 记录值 |
|---|---|---|
| CNAME | `www` | `flowerwithwind.github.io` |

访问：`https://www.你的域名.com`

（可选）根域名跳到 www：用注册商提供的「URL 转发」或 Cloudflare Page Rule。

### 方案 B：根域名（apex，如 `你的域名.com`）

GitHub 要求 **A 记录**（或 ALIAS/ANAME，若服务商支持）：

| 类型 | 主机记录 | 记录值 |
|---|---|---|
| A | `@` | `185.199.108.153` |
| A | `@` | `185.199.109.153` |
| A | `@` | `185.199.110.153` |
| A | `@` | `185.199.111.153` |

（IP 以 [GitHub 文档](https://docs.github.com/en/pages/configuring-a-custom-domain-for-your-github-pages-site/managing-a-custom-domain-for-your-github-pages-site) 最新列表为准。）

也可同时加：

| 类型 | 主机记录 | 记录值 |
|---|---|---|
| CNAME | `www` | `flowerwithwind.github.io` |

---

## 四、验证

```bash
# 应返回你的域名
nslookup www.你的域名.com

# 打开浏览器
https://www.你的域名.com/
https://www.你的域名.com/projects/
```

DNS 全球生效常见 **10 分钟～48 小时**。  
GitHub 的 HTTPS 证书要等 DNS 正确后再点 Enforce HTTPS。

---

## 五、绑定后路径对照

| 原来 | 绑定 www 后 |
|---|---|
| `...github.io/personal-dev-website/` | `https://www.你的域名.com/` |
| `.../projects/` | `https://www.你的域名.com/projects/` |
| `.../notes/` | `https://www.你的域名.com/notes/` |

仓库名可以不改；自定义域名下站点 `base` 会变成 `/`，不再带 `personal-dev-website` 前缀。

---

## 常见问题

**证书一直失败**  
先确认 DNS 已指向 GitHub，再取消并重新勾选 Enforce HTTPS。

**打开是 404**  
确认 Pages Source 是 `gh-pages` 分支根目录，且最近一次 deploy 成功。

**想换回 github.io**  
把 `customDomain` 改回 `''`，重新 build + deploy，并在 Pages 设置里删掉 Custom domain。

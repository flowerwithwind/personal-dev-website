Code Review Agent

面向 GitHub Pull Request 的智能代码评审 Agent。自动解析 Diff，结合 RAG 规范与历史经验，完成多维度审查并生成结构化报告与 PR 评论。

状态可本地 / Docker 部署
目标岗位映射Code AI / Coding Agent

01 · 问题背景

人工 Code Review 成本高、反馈慢，且质量依赖评审者经验。现有「只贴 Diff 给大模型」的方式存在上下文不足、规范不一致、token 失控与无法接入协作流程等问题。

本项目要回答：如何把 LLM 审查做成可触发、可增量、可回写 GitHub、可被规范知识增强的工程系统，而不是一次性 Demo 脚本。

02 · 解决方案

采用「Webhook / API / CLI 多入口 + 审查引擎 + RAG 知识增强」的分层架构：

GitHub PR Event → Webhook 校验 → Diff 解析 / 语言过滤 → 分块 & 增量审查（控 token） → RAG：规范 / 历史 Review / 最佳实践 → LLM 多维度评分与建议 → 结构化报告 + PR Comment → 任务状态可查询

03 · 我的贡献

  • 设计审查流水线：Diff 解析、语言感知过滤、分块增量策略，控制成本与覆盖面。
  • 实现 RAG 增强链路：规范文档、历史评审与最佳实践检索，提高建议可操作性。
  • 打通 GitHub Webhook、REST API 与 CLI，支持本地调试与服务化部署。
  • 输出结构化报告模板与 PR 评论，使结果可直接进入协作流程。
  • 补充 Docker / conda 环境约定与测试夹具，降低复现成本。

04 · 技术栈

Python FastAPI LLM Agent RAG ChromaDB DeepSeek GitHub API Docker Pytest

05 · 成果与亮点

多维度审查 正确性 / 安全 / 性能 / 可维护性 / 风格，输出结构化结果而非空泛摘要。
工程闭环 从触发、排队、审查到评论回写形成完整链路，支持服务化运行。
成本可控 增量与分块策略减少无关上下文，适配真实 PR 体量。
可演示可部署 本地 conda、脚本与 Docker Compose 多路径启动，便于面试现场讲解。

06 · 反思与下一步

  • 可增强：仓库级规则配置、语言插件化、评审质量在线评估。
  • 可增强：与 CI 门禁结合（仅对高风险问题阻断）。
  • 叙事建议：面试中用一次真实 PR Diff 走完整链路，展示可观测日志与报告。
